## 项目背景
- **目标系统**：用户上传图片 → 本机OCR识别文字 → 调用外部LLM API进行文字替换（翻译/改写/脱敏等）→ 将新文本重绘到原图 → 返回结果
- **部署环境**：单机4核8G纯CPU服务器（轻量级，非高并发）
- **技术栈**：Python + FastAPI + PaddleOCR + OpenCV/PIL + 外部LLM API（如通义千问、GPT等）
- **预期QPS**：5~10（每个cpu服务器中低负载）

| **后端开发工程师** | 1~2 | 实现FastAPI服务、OCR集成、LLM API调用、图像重绘逻辑、接口文档、日志监控、部署脚本 | 必需 | 核心开发，需熟悉Python异步框架、HTTP、图像处理 |

| **UI/UX设计师** | 0~1 | 设计上传页面、反馈提示、错误状态样式（如“识别失败”、“处理中”） | 可选 | 若有Web界面，建议有设计感；纯API可省略 |
| **数据标注员 / 数据准备** | 0~1 | 准备测试用图片集（含不同字体、背景、角度、语言），用于优化OCR效果 | 可选 | 如需提升识别准确率，需人工标注训练/评估样本 |
| **DBA（数据库管理员）** | 0 | 本项目无持久化数据库需求 | 不需要 | 所有数据为临时处理，无需存储历史记录（除非有审计要求） |

## 角色能力要求

### 后端开发（关键角色）
- 精通 Python（FastAPI / Flask）
- 熟悉异步编程（async/await）
- 掌握图像处理库：PIL / OpenCV
- 熟悉 OCR 工具：PaddleOCR / EasyOCR（能调参优化识别精度）
- 会调用 RESTful API（LLM 接口：鉴权、重试、超时控制）
- 了解 Docker 基础（打包镜像）
- 有基础性能调优经验（避免内存泄漏、长连接超时）
