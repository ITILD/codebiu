### 一、典型软件开发流程

- **需求分析阶段**
- **项目规划阶段**
- **设计阶段**
- **开发阶段**
- **测试阶段**
- **部署上线阶段**
- **维护与迭代阶段**

---

### 二、阶段资料

#### 1. 需求分析阶段

明确项目要做什么，需求是什么。

- **需求文档（SRS, Software Requirements Specification）**
  - 功能需求（用户能做什么）
  - 非功能需求（性能、安全性、可用性等）
- **用户调研报告 / 用户访谈记录**
- **业务流程图 / 用例图（Use Case Diagram）**
- **原型设计图（如 Axure、Figma 设计稿）**
- **需求评审会议纪要**
---

#### 2. 项目规划阶段

制定项目计划，分配资源，估算时间与成本。
- **项目立项书 / 项目建议书**
- **项目计划书（Project Plan）**
  - 里程碑（Milestones）
  - 时间表（甘特图）
- **资源分配表（人员、设备等）**
- **风险评估报告**
- **预算估算文档**
- **WBS（工作分解结构）**
---

#### 3. 设计阶段

需求转化为技术方案。

- **系统架构设计文档**
  - 技术选型（前端、后端、数据库、中间件等）
  - 架构图（如微服务、单体、分层架构）
- **数据库设计文档（ER 图、表结构）**
- **API 接口设计文档（如 Swagger/OpenAPI）**
- **UI/UX 设计稿（高保真原型）**
- **安全设计文档（如权限模型、加密策略）**
- **非功能性设计（如高可用、容灾）**
---

#### 4. 开发阶段

编写代码，实现功能。

- **代码仓库（如 Git）**
  - 分支管理策略（Git Flow）
- **编码规范文档**
- **开发环境配置说明（如 Dockerfile、.env 示例）**
- **模块划分文档 / 接口对接说明**
- **每日/每周开发进度报告**
- **代码评审记录（Code Review）**

---

#### 5. 测试阶段

验证软件是否符合需求和质量标准。

- **测试计划文档**
- **测试用例（Test Cases）**
- **测试数据**
- **缺陷报告（Bug Report）**
  - 使用 Jira、禅道等工具管理
- **自动化测试脚本（如 Selenium、JUnit）**
- **测试报告（Test Summary Report）**
  - 覆盖率、通过率、遗留问题等

---

#### 6. 部署上线阶段

将系统部署到生产环境并对外提供服务。

- **部署文档（Deployment Guide）**
  - 环境配置、依赖安装、启动命令
- **上线检查清单（Checklist）**
- **回滚方案（Rollback Plan）**
- **运维手册 / 操作手册**
- **监控与日志配置（如 Prometheus、ELK）**
- **发布通知（给用户或内部团队）**

---

#### 7. 维护与迭代阶段

修复问题、优化性能、增加新功能。

- **用户反馈记录**
- **运维日志 / 错误日志**
- **版本更新日志（Changelog）**
- **迭代需求文档（新功能或优化）**
- **定期维护计划**

---

### 四、推荐工具（辅助资料管理）

| 类型     | 工具示例             | 备注                                          |
| -------- | -------------------- | --------------------------------------------- |
| 文档协作 | 飞书文档             | 短期文档用飞书/微信/内网通信软件 长期用 git   |
| 代码管理 | Git、GitLab、Gitea   | GitLab 为主                                   |
| 原型设计 | Figma、Axure         | Figma 为主，出色系/字体/页面样式/简单交互逻辑 |
| 接口文档 | Swagger、Postman     | Swagger                                       |
| 测试管理 | TestRail、Zentao     | 不是很了解                                    |
| 部署运维 | Jenkins、Docker、K8s | 开发一般出 Docker 配置书为主                  |

---

### 总结：关键资料清单

| 阶段 | 核心资料                     | 备注                              |
| ---- | ---------------------------- | --------------------------------- |
| 需求 | 需求文档、原型图、用例图     |                                   |
| 规划 | 项目计划书、WBS、风险评估    |                                   |
| 设计 | 架构图、数据库设计           | 简稿，实际图可以后生成            |
| 开发 | 代码、开发规范、接口说明     | 自动生成的 swagger 和必要接口注释 |
| 测试 | 测试报告                     | 重要或复杂模块有单测              |
| 部署 | 版本部署文档                 |                                   |
| 维护 | 变更记录、用户反馈、更新日志 |                                   |

---

**文档服务于沟通与协作，不必追求完美，但需清晰、可追溯!!!**。
